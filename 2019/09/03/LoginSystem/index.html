  • 简介

    • ！纯练习使用，切勿用于生产环境 ！
    • 基于之前练习的一个基于jsp、jdbc、servlet、mysql的一个crud项目，想深入练习一下，做一个什么什么系统什么的，但基本上是个系统就会有登录功能，那就从登录功能开始实现吧！
    • 经过大概两周的断断续续的练习，注册，登录，修改，注销功能均已实现。

  • 设计

    • 登录系统应当可以 注册、登录、修改密码、以及注销用户
    • 界面使用最简单的html实现，不求美观，实现功能即可
    • 数据库 test_db 表 user 中有 userName 和 password 两列，其中userName为主键
    • 类 User user.java 中有 userName 和 password 变量 以及变量对应的get和set方法
    • 数据访问层 UserDao.java 连接数据库，并实现对数据库中数据的增删改查四个操作
    • 控制+业务层 UserServlet.java 接收页面传来的参数，并且调用Dao中的增删改查来实现 登录、注册、修改密码、删除四个操作
    • 页面 登录页面 index.jsp 注册页面 register.jsp 用户页面 user.jsp 修改页面 modify.jsp

  • 解析

    • 开发环境：IDE：idea 2019.1.3、Java：jdk11、server Tomcat 8.5 mysql 5.5
    • 技术栈 HTML、javascript、java、servlet、jdbc
    • 相对于基本的增删改查，一个基本的登录系统增加了用户名密码是否为空、用户名是否存在、密码是否正确等校验，页面的跳转，以及参数在页面中传递等技术细节。

  • 总结：从最初的想法，到最后系统的完成，细节方面在开发与测试中逐渐变得清晰，比如原来并没有想到的用户名不为空但不存在与数据库中的情况等，具体细节的完善可以参考设计流程图与完成之后的流程图，系统的实现并没有参考其它登录系统，完全按照自己的想法来完成，参数传递部分给登录系统带来很大的安全问题，页面也不够美观，系统里有一些用到的代码也不懂，但这只是一个开始，目标是迭代出一个最够安全，易用，美观的登录系统模组。具体部署的细节与环境的配置自行google,或许以后会写一写这方面的文章，或许不会
